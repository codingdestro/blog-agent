About the Project

From prompt to publish-ready article

The AI Blog Generator is a full-stack application that leverages transformer-based LLMs combined with retrieval-augmented generation (RAG) to produce factually grounded, well-structured blog posts. Users provide a topic and tone — the system handles research, drafting, and SEO optimisation.

The project explores hallucination mitigation, source citation, and the quality-speed tradeoff in automated content pipelines — key open problems in applied NLP.

async def generate_blog(topic, tone):
  sources = await web_search(topic)
  context = rag_retrieve(sources)
  outline = llm_plan(topic, context)
  draft  = await llm_write(outline, tone)
  seo    = seo_score(draft)

  return {
    "article": draft,
    "sources": sources,
    "seo_score": seo
  }
Core Capabilities

Built for quality, not just speed

🔍

Real-Time Web Retrieval

Fetches live search results and ingests authoritative sources before writing, grounding every claim in verifiable data.

🧠

RAG Pipeline

Retrieval-Augmented Generation embeds and ranks source chunks, feeding the model only the most relevant context — reducing hallucination.

✍️

Tone Control

Switch between formal, conversational, technical, and promotional registers. The model adapts vocabulary and sentence structure accordingly.

📈

SEO Optimisation

Keyword density analysis, heading hierarchy scoring, and readability checks ensure articles rank well on search engines.

🔗

Auto Citation

Every factual claim is linked back to its source. Citations are formatted and injected automatically at the end of each section.

⚡

Streaming Output

Token-by-token streaming renders the article in real time, providing immediate feedback and dramatically reducing perceived latency.

Methodology

Four-stage generation pipeline

01

Input & Config

User enters a topic, preferred tone, target length, and any keywords to include or avoid.

02

Research & Retrieval

Live web search + vector similarity retrieval surfaces the most relevant and recent sources.

03

LLM Drafting

The language model drafts title, outline, sections, and summary using retrieved context as grounding.

04

Post-Processing

SEO scoring, source linking, readability check, and final formatting before delivery.
